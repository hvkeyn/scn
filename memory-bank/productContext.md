# Product Context: LocalSend

## Проблема, которую решает проект

LocalSend решает проблему передачи файлов между устройствами без использования облачных сервисов, интернета или внешних серверов. Это особенно важно для:

- **Приватности** - данные не проходят через сторонние серверы
- **Скорости** - использование локальной сети обеспечивает высокую скорость передачи
- **Надежности** - работа без зависимости от интернет-соединения
- **Безопасности** - прямое соединение между устройствами с TLS шифрованием

## Целевая аудитория

- Пользователи, которым нужна быстрая передача файлов между устройствами
- Люди, заботящиеся о приватности данных
- Пользователи в средах без стабильного интернета
- Разработчики и технически подкованные пользователи

## Как это должно работать

### Основной сценарий использования

1. **Обнаружение устройств**
   - Приложение запускается на всех устройствах
   - Используется multicast DNS для обнаружения других устройств в локальной сети
   - Устройства отображаются в списке доступных получателей

2. **Отправка файлов**
   - Пользователь выбирает файлы для отправки
   - Выбирает получателя из списка доступных устройств
   - Файлы передаются по HTTPS с TLS шифрованием
   - Отображается прогресс передачи

3. **Получение файлов**
   - Приложение работает в режиме получения
   - Показывает входящие запросы на передачу
   - Пользователь может принять или отклонить передачу
   - Полученные файлы сохраняются в указанную папку

### Дополнительные функции

- **История** - сохранение истории полученных файлов
- **Избранное** - быстрый доступ к часто используемым устройствам
- **Настройки** - настройка портов, шифрования, папки для сохранения
- **Темы** - поддержка светлой, темной и OLED тем
- **Мультиязычность** - поддержка 106 языков

## Пользовательский опыт

### Принципы UX

1. **Простота** - минимальные настройки, работа "из коробки"
2. **Прозрачность** - понятный интерфейс, видимый прогресс передачи
3. **Надежность** - стабильная работа, обработка ошибок
4. **Скорость** - быстрая передача, минимальные задержки
5. **Безопасность** - визуальные индикаторы безопасности, контроль доступа

### Основные экраны

- **Receive Tab** - экран получения, показывает статус и входящие запросы
- **Send Tab** - экран отправки, выбор файлов и получателей
- **Settings Tab** - настройки приложения

## Технические ограничения

- Требуется локальная сеть (Wi-Fi или Ethernet)
- Необходимо отключить AP isolation на роутере
- Порт 53317 должен быть открыт для входящих соединений
- На Windows сеть должна быть настроена как "частная"

## Конкуренты и отличия

- **AirDrop (Apple)** - только для устройств Apple
- **Nearby Share (Google)** - только для Android
- **Snapdrop** - требует интернет-соединение
- **LocalSend** - кроссплатформенный, работает без интернета, open-source
